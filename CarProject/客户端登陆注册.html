客户端登陆注册

在处理约车业务之前，需要司机和乘客注册帐号，或者使用已有账户进行登陆。为了实现注册和登陆功能，整个框架的各个模块需要提供相应功能。

处理流程

  • 客户端：提供输入界面
  • 应用服务器：进行逻辑处理，包括判断该用户是否存在，验证账户信息等
  • 数据服务器：保存用户信息

接口

  • 应用服务器：http://app-server/user
  • 数据服务器：http://data-server/sql

    cmd: "reg",

    mobile: "13312345678"

应用服务器到客户端

  • 成功时返回格式 
{
    cmd: "regack",
    result: "ok",
    userid: "userid",
    sessionid: "sessionid"
}

应用服务器到数据服务器

{
    sql: "sql语句"
}

数据服务器到应用服务器

  • 成功时返回 
{
    result: "ok",
    data:{

    }
}
  • 失败时返回 
{
    result: "error",
    reason: "why error"
}
