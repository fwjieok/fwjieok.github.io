乘客发送订单请求

当用户需要约车时，通过界面发送约车请求。

约车请求发送给应用服务器，应用服务器在处理完毕之后，发送给数据服务器，在数据服务器生成一个订单。

处理流程

  • 客户端：提供输入框、供用户输入开始位置和目标位置
  • 应用服务器：对客户的请求进行处理
  • 数据服务器：保存订单

接口

  • 应用服务器：http://app-server/order
  • 数据服务器：http://data-server/sql

数据报文

客户端到应用服务器：

  • 请求报文 
{
    cmd: "orderrequest",
    sessionid: "sessionid",
    startpos: {
        longitude: 120.010203,
        latitude: 45.010203,
        name: "北京清河中街"
    }
    endpos:
    {
        longitude: 120.010203
        latitude:  45.010203
        name: "北京天安门"
    }
}

应用服务器到客户端

应用服务器收到订单请求后，查找空闲的附近的司机，并将其纪录在订单中。

  • 处理成功响应报文 
{
    cmd: "orderrequest",
    result: "ok",
    orderid: "orderid"
}
  • 处理失败响应报文 
{
    cmd: "orderrequest",
    result: "error",
    reason: "something wrong..."
}
